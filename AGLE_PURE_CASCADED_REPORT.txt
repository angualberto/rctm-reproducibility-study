2. ARQUITETURA AVALANCHE ENCADEADO
================================================================================

ANTES (Linear com 1 avalanche final):
──────────────────────────────────────
  pool[i] + chaos_bits → avalanche → saída
  
  Problema: Soma é LINEAR, bits podem cancelar

AGORA (Avalanche Encadeado - 3 camadas):
──────────────────────────────────────
  pool[i]
     ↓
  avalanche(pool[i] ⊕ rotl(pool[i+1], 17))
     ↓
  avalanche(step1 ⊕ rotl(pool[i+7], 41))
     ↓
  avalanche(step2 ⊕ chaos_bits)
     ↓
  saída

  Vantagem: DIFUSÃO NÃO-LINEAR EM CASCATA

3. ESTRUTURA DO CÓDIGO

3.1 update_entropy_pool() - 3 Camadas de Avalanche
───────────────────────────────────────────────────

static void update_entropy_pool(uint64_t *chaos_bits)
{
    for (int i = 0; i < AGLE_POOL_SIZE; i++) {
        uint64_t a = pool[i];
        uint64_t b = pool[(i+1) % AGLE_POOL_SIZE];
        uint64_t c = pool[(i+7) % AGLE_POOL_SIZE];
        uint64_t d = chaos_bits[i % 5];

        /* Camada 1: Mistura com vizinho próximo */
        uint64_t step1 = avalanche64(a ^ rotl(b, 17));
        
        /* Camada 2: Mistura com vizinho distante */
        uint64_t step2 = avalanche64(step1 ^ rotl(c, 41));
        
        /* Camada 3: Injeção de caos */
        uint64_t step3 = avalanche64(step2 ^ d);

        pool[i] = step3;
    }
}

Impacto:
  ✓ Cada palavra do pool passa por 3 transformações não-lineares
  ✓ Depende de 2 vizinhos + caos
  ✓ Elimina possibilidade de reversão linear

3.2 mix_pool_cross() - 3 Camadas de Avalanche × 8 Rodadas
──────────────────────────────────────────────────────────

static void mix_pool_cross(void)
{
    for (int round = 0; round < 8; round++) {
        for (int i = 0; i < AGLE_POOL_SIZE; i++) {
            uint64_t a = pool[i];
            uint64_t b = pool[(i+1) % AGLE_POOL_SIZE];
            uint64_t c = pool[(i+7) % AGLE_POOL_SIZE];
            uint64_t d = pool[(i+11) % AGLE_POOL_SIZE];

            /* Avalanche encadeado: 3 camadas */
            a = avalanche64(a ^ rotl(b, 17));
            a = avalanche64(a ^ rotl(c, 41));
            a = avalanche64(a ^ rotr(d, 13));

            pool[i] = a;
        }
    }
}

Impacto:
  ✓ 8 rodadas × 3 camadas = 24 transformações avalanche por palavra
  ✓ Total: 16 palavras × 24 = 384 aplicações de avalanche
  ✓ Difusão extrema em todo o estado

3.3 update_accumulator() - 4 Camadas de Avalanche
──────────────────────────────────────────────────

static void update_accumulator(uint64_t jitter)
{
    /* Camada 1: acc + pool[0] */
    uint64_t mix1 = avalanche64(acc ^ pool[0]);
    
    /* Camada 2: mix1 + pool[5] */
    uint64_t mix2 = avalanche64(mix1 ^ rotl(pool[5], 23));
    
    /* Camada 3: mix2 + pool[10] */
    uint64_t mix3 = avalanche64(mix2 ^ pool[10]);
    
    /* Camada 4: mix3 + jitter + contador */
    acc = avalanche64(mix3 ^ jitter ^ (generation_counter * 0x94d049bb133111ebULL));
}

Impacto:
  ✓ 4 camadas de avalanche em série
  ✓ Acumulador completamente irreversível
  ✓ Injeta jitter e contador a cada geração

4. FLUXO COMPLETO DO RNG
================================================================================

uint64_t agle_pure_next(void)
{
    1. Atualizar contadores
       generation_counter++
       sensor_counter++

    2. Injetar ruído físico
       jitter = clock_jitter()  (rdtsc + CLOCK_MONOTONIC)

    3. Atualizar mapas caóticos acoplados
       x → logistic(x + y * 0.1)
       y → tent(y + temp * 0.1)
       temp → logistic(temp + fan * 0.1)
       fan → tent(fan + volt * 0.1)
       volt → chaos_step(volt + x * 0.1)

    4. Extrair bits dos erros de ponto flutuante
       chaos_bits[5] ← mantissas + erros de quantização

    5. Injetar sensores físicos (periódico)
       chaos_bits[0-2] ^= sensor_cache.{temp, fan, volt}

    6. Atualizar pool (AVALANCHE ENCADEADO × 3)
       pool[i] → avalanche → avalanche → avalanche

    7. Mistura cruzada (AVALANCHE ENCADEADO × 3 × 8 rodadas)
       384 aplicações de avalanche total

    8. Atualizar acumulador (AVALANCHE ENCADEADO × 4)
       acc → avalanche → avalanche → avalanche → avalanche

    9. Gerar saída
       out = pool[0] ^ pool[3] ^ pool[7] ^ pool[11] ^ acc
       return avalanche(out)
}

6. COMPLEXIDADE COMPUTACIONAL
================================================================================

Por geração de uint64_t:
  • 3 avalanche (update_entropy_pool)
  • 384 avalanche (8 rodadas × 3 camadas × 16 palavras)
  • 4 avalanche (update_accumulator)
  • 1 avalanche (generate_output)
  • 5 atualizações de mapas caóticos
  • 1 clock_jitter

Total: ~392 chamadas de avalanche64 por uint64_t gerado
       + 5 operações de ponto flutuante

Complexidade: O(1) constante, mas com alta difusão

================================================================================
7. ESTADO INTERNO
================================================================================

Total de bits de estado:
  • pool[16]: 16 × 64 = 1024 bits
  • state_x, state_y, state_temp, state_fan, state_volt: 5 × 64 = 320 bits FP
  • acc: 64 bits
  • generation_counter: 64 bits
  • sensor_counter: 32 bits
  • sensor_cache: 3 × 64 = 192 bits

TOTAL: 1024 + 320 + 64 + 64 + 32 + 192 = 1696 bits de estado

Estimativa de período:
  • Estado integer: 2^1024 (pool)
  • Estado caótico: não periódico (erros FP)
  • Período teórico: > 2^1024
